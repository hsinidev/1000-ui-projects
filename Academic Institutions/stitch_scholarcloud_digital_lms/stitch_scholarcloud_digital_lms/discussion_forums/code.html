Best practices for documenting Large Scale API architectures?

Alex Rivera • Posted in Engineering • 2 hours ago

Hi everyone! I'm currently working on my capstone project which involves a microservices architecture with over 15 distinct internal APIs. I'm finding it difficult to maintain documentation that stays in sync with code changes.

Has anyone had success with Swagger/OpenAPI auto-generation in a CI/CD pipeline? Or are there better tools you'd recommend for an academic team environment?

Sarah Jenkins 1h ago

We used the Swagger-UI plugin for our Jenkins pipeline last semester. It works quite well if you're consistent with your docstrings. Just make sure your build fails if the schema doesn't validate!

Marcus Chen 45m ago

Agreed on the validation. We actually added a spectral linter to our OpenAPI specs to ensure we weren't missing descriptions or response codes.
