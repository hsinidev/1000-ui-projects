Authentication Protocol

The Forge Protocol utilizes high-entropy JWT tokens combined with hardware-backed security modules. This guide details the handshake process required for establishing expert-grade connections with our API clusters.

GET /auth/v1/session

Retrieve current session status and telemetry data for the active node cluster.

POST /auth/v1/token

Exchange valid developer credentials for an ephemeral bearer token.

cURL Authentication 
curl -X POST https://api.forge-protocol.com/v1/auth \
  -H "Content-Type: application/json" \
  -d '{
    "client_id": "YOUR_DEV_ID",
    "secret": "FORGE_SECRET_KEY",
    "scope": "read:analytics"
  }'
